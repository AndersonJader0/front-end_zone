Comunicação Cliente Servidor Cliente: Somos nós, quando abrimos nosso browser e pesquisamos algo, abrimos uma página ... Consumindo alguma informação. Servidor: Uma máquina conectada na internet por 24h por dia que irá servir a informação para o cliente. (Hoje temos servidores na nuvem, virtualizações...) Cliente quer ver a página http://andersonjader.com.browser É enviado um pedido, requisição, request ao servidor. Será verificado que servidor tem essa página. Servidor vai interceptar esse pedido e retornar(response) uma cópia da página HTML em forma de texto que aparecerá normal pro cliente. Fundamental: URL - Localizador de recursos (endereço de um arquivo, portanto, único) HTTP - Protocolo de transferência de Hipertexto(Padrão de comunicação entre diferentes máquinas imdependente dos servidores, SO, hardware, software etc) HTML - Linguagem de marcação de Hipertexto (Uma maneira de termos um arquivo vinculado) Programação exige lógica de programação e portanto, não programamos em HTML mas asim marcamos em HTML.
